Live deployment path

Where the platform runs today and what handover looks like.
Primary server
New Contabo VPS
OR migrate to client-provided server
Hosts the EW Flask app, Postgres, Nginx/SSL, FlareSolverr, cron scanners. Either we spin up a fresh Contabo box in the client's name (~$50/mo) or we migrate the stack to a server they already own. One-time migration: roughly half a day of work.
Backup / failover server
Second VPS, cold standby
Identical configuration, hourly DB snapshot replication. If the primary server dies, DNS flips and EW is back online in minutes. ~$50/mo parked in standby.

Local mini-servers (Beelinks)

Small residential-IP boxes running on the client's local network. Required because vAuto (Cox) binds sessions to residential IPs and rejects datacenter traffic. One-time cost each, no monthly fee beyond electricity.
Mini-server role Purpose Hardware cost
vAuto verifier + new-dealer onboard Runs the days-on-lot rBook verification for every car across every active partner dealer. Automatically picks up new cars nightly and processes any newly-added dealer the next morning. One Chrome session, one Cox login, residential IP. ~$300
Bid-listing background worker (A) Processes incoming bids as they land in the EW dashboard: fires the Gemini assessment, pulls Carfax/AutoCheck via vAuto, grabs AccuTrade + iPacket screenshots, runs OCR fallbacks. Handles live bid traffic. ~$300
Bid-listing background worker (B) — redundant Second identical worker as a hot partner to worker A. Load-balances bid processing during peak hours; takes over automatically if A fails. Also provides capacity headroom as dealer count + bid volume grow. ~$300
TOTAL local mini-servers ~$900 one-time
All three Beelinks are one-time hardware purchases. No recurring fees. Run Ubuntu + systemd services, auto-recover on reboot, no daily touch required after initial Okta login.
